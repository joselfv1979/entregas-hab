Aplicación REST dividida en varios módulos: el archivo principal que ejecuta la aplicación importa desde users.js el módulo de login que realiza la autenticación, en este mismo archivo hay un usuario creado previamente 'admin' con su password encriptada, el password en claro es 'nimda'. El archivo products.js exporta los módulos para listar, añadir y modificar productos. El archivo auth.js contiene el módulo que comprueba la autorización para poder añadir y modificar a través del token entregado al loguearse, que debe añadirse en el campo 'authorization' de los headers de la petición correspondiente (la acción de listar no requiere autenticación previa). 
Por último el archivo .env es un archivo de configuración que guarda las variables de entorno que utiliza la aplicación.
